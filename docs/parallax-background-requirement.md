背景圖片縮放策略：
1. **優先原則**：確保圖片寬度與容器寬度一致 (Width = Container Width)，並在維持原始比例的前提下計算圖片高度。
2. **例外處理 (覆蓋機制)**：若依上述原則計算出的圖片高度 *小於* 視窗高度 (Viewport Height)，則強制將圖片高度設定為視窗高度 (Height = Viewport Height)，並依比例放大寬度 (此時寬度會大於容器寬度，呈現 overflow 狀態)。此機制是為了防止在長型螢幕 (如 Mobile) 上出現垂直留白。

位移 (Parallax) 邏輯：
1. **位移量計算**：`TargetShift = Viewport Height - Image Height`。
2. **捲動對應**：
   - 頁面頂部 (Scroll 0%) -> 背景頂部對齊視窗頂部 (y = 0)。
   - 頁面底部 (Scroll 100%) -> 背景底部對齊視窗底部 (y = TargetShift)。
3. **速度變化**：
   - 當圖片比視窗長時 (常見情況)：背景會隨捲動向上移動 (速度較慢，產生視差)。
   - 當圖片高度強制等於視窗高度時 (例外情況)：`TargetShift = 0`，背景將固定不動，確保至少填滿畫面。

其他限制：
- 背景圖片必須無縫填滿整個視窗區域 (Viewport)，不得出現白邊或破圖。
- 容器必須設定為 `fixed` 定位，避免因頁面長度變化或 `absolute` 定位導致的起始點偏移。
- 嚴格禁止出現因計算誤差導致的水平偏移或是左側留白 (Mobile X必須為0)。
